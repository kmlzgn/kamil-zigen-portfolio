Case Study P7 · Error-Proofing / Quality Engineering
BorgWarner İzmir · 2017
P7 · BorgWarner · Audi V6 HPS · CNC Gundrilling · Op 20

PLC-Based Poka-Yoke
— Audi V6 HPS

Process Leader · 3 Months · RCA (5 Whys / Ishikawa) · PFMEA · PLC Error-Proofing · PPAP
Business Challenge
Recurring depth deviation in Audi V6 HPS components during CNC gundrilling at Op 20. Parts were over-machined when the HPV CNC program was applied to HPS parts at changeover — a 16.5 mm depth parameter mismatch (−213.0 mm vs. required −196.5 mm). A second concurrent failure: incorrect gage used during setup verification, preventing detection. Both failure modes were human-dependent. Both had recurred despite procedural containment applied twice previously.
Root Cause 1 — CNC Program Mismatch (Occurrence) · Before vs. After
✗ Before — HPV Program
✓ After — HPS Program
Real Op 20 part photos and CNC editor code — HPV program (Δ Phase 2 = −213.0 mm) applied to an HPS part vs. correct HPS program (−196.0/−196.5 mm). Δ = 16.5 mm over-machining, invisible on the shop floor without a system check.
Root Cause 2 — Gage Selection Error (Detection Failure)
The incorrect HPV gage (red) is visually near-identical to the correct HPS gage (green) — no system flag existed to catch the mismatch during setup verification.
System Solution — PLC Poka-Yoke Logic
Coded PLC connector — physical fixture/program interlock, no operator bypass possible
HMI screen — part/program/gage 3-way match confirmed, cycle start authorised
Implementation
  • Fixture socket interface: physical connector reads part type automatically at load — no operator input required
  • 3-way PLC validation: part reference → CNC program → gage must all match before cycle start is authorised
  • Machine interlock: any mismatch locks machine; supervisor acknowledgement required to clear
  • No bypass: validation sequence cannot be overridden by operator action under any circumstance
  • PFMEA closure: occurrence and detection ratings both reduced to minimum level
  • PPAP closed — updated control plan validated and approved
Why This Approach
Procedural controls — checklists, operator training, visual reminders — had been applied twice and had failed. When a failure mode carries direct customer consequence, the countermeasure must remove the human decision point at the machine level. PLC-level validation is the only approach that achieves this reliably.

"The only way to guarantee zero recurrence was to make a wrong setup physically impossible to execute. That required a system solution, not a procedural one."

Case Study P4 · Delivery Performance
Trio Composites İzmir · 2022–2023
P4 · Trio Composites · Plant Manager

On-Time Delivery Recovery
70.3% → 94.8%

Plant Manager · 4 Months · Lean Scheduling · KPI System Design · Cross-Functional Coordination · Buffer Management
Business Challenge
OTD stagnated at 70.3% average across H1 2022 — 20 percentage points below the 90% customer target. Planning was entirely reactive: problems were managed after delays had already occurred. No shared visibility across production, logistics, and procurement. High expediting costs and deteriorating customer relationships.
Root Cause
  • Procurement delays not visible to production until after production start date
  • Scheduling with no buffer — demand variation translated directly to delivery misses
  • No cross-functional escalation trigger before the due date was missed
Implementation
  • Proactive dashboard: at-risk orders flagged ≥5 working days before due date, visible to all functions simultaneously
  • Daily 15-min stand-up: cross-functional review with defined escalation
  • Buffer management: scheduling absorbs demand variation before it becomes a miss
  • Supplier early warning: material delays trigger rescheduling, not expediting
  • Quality review cadence established — daily shopfloor reviews and RCA routines embedded

Key Learning
OTD problems almost never come from capacity. They come from information arriving too late to act on. The dashboard did not add capacity — it moved the decision point five days earlier. That alone was enough to recover 25 percentage points.
The harder lesson: information without a decision-making forum changes nothing. The daily stand-up was as important as the dashboard.

"The plant did not need more people or more capacity. It needed to know about problems five days earlier."
